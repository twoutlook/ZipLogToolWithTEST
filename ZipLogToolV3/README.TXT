=== Framework 4.7 命令行壓縮工具 ===

需求規格說明:
1. 在 .NET Framework 4.7 環境中，開發一個用於壓縮指定路徑的命令行工具。
2. 使用類似 INI 格式的配置文件來列出多個要處理的目錄。
3. 在執行時，根據系統時間，檢查資料夾及檔案的名稱，其命名規則為 yyyy-MM-dd{可包含附加文字}{如果是檔案，限定為.log或.txt}，處理上個月份及更早的資料夾及檔案，並將它們按月份壓縮成單獨的 ZIP 文件。
4. 例如，在9月12日運行時，所有 2024-08-05、2024-08-06 等的資料夾以及 2024-08-05.log、2024-08-05.txt 等檔案都會被壓縮成一個名為 2024-08.zip 的文件。
5. 如果特定月份的 ZIP 文件已存在，但在後續檢查中發現了新內容，例如在9月13日再次檢查時發現了新的 2024-08-25.log 或 2024-08-26.log，則這些新內容將被壓縮成一個新的 ZIP 文件，並添加後綴，命名為 2024-08_1.zip。
6. 若在9月14日再次檢查，仍有新內容（如 2024-08-27.log），則將其壓縮成另一個新的 ZIP 文件，並繼續增加後綴數字，例：2024-08_2.zip。
7. 成功壓縮後，將原始文件和資料夾刪除，以保持目錄整潔。
8. 在工作目錄中生成日誌文件，詳細記錄所有操作，包括讀取的 INI 文件內容、檢查到的目錄和檔案、邏輯判斷過程、執行的操作，以及成功壓縮後刪除原始文件的記錄等。
